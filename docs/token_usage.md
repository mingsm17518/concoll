# Token 消耗估算

## 数据集信息

- **数据集**: PrimeVul test_paired
- **配对样本数**: 435
- **代码样本总数**: 870 (vulnerable_code + fixed_code)
- **平均代码长度**: 5,306 字符

## Token 消耗计算

使用 logprobs 计算置信度，基于置信度阈值自动分类。

### 情况 1: 典型分布 (70% Stage 1, 25% Stage 2, 5% Stage 3)

| 阶段 | 样本数 | Prompt Tokens | Completion Tokens | 总计 |
|------|--------|---------------|-------------------|------|
| Stage 1 (Direct) | 870 | 611,610 | 1,740 | 613,350 |
| Stage 2 (RAG) | 217 | 232,841 | 434 | 233,275 |
| Stage 3 (Multi-Agent×3) | 43 | 97,782 | 258 | 98,040 |

**总计**:
- Prompt Tokens: 942,233
- Completion Tokens: 2,432
- **Total: 944,665**

### 情况 2: 最坏情况 (所有样本都经过 Stage 3)

当置信度阈值设置过高时（如 th1=0.99），所有样本可能都被拒绝到后续阶段。

| 阶段 | 样本数 | Prompt Tokens | Completion Tokens | 总计 |
